## Troubleshooting

### Build fails with "DATABASE_URL not set"
- Make sure you added the `DATABASE_URL` environment variable in Render dashboard BEFORE deploying
- Re-deploy after adding the env var

### Images not loading
- Render serves static files automatically because they're in the `/static` folder at project root
- If images are broken, check that the image filenames match exactly (they're case-sensitive on Linux servers)

### Database connection errors
- **"Name or service not known"**: This was caused by a bug in `db.py` that stripped the regional subdomain (e.g., `aws-1-ap-southeast-1.`) from the hostname while fixing the `@` in the password. This has been fixed by using `urllib.parse` for robust URL encoding.
- Verify your Supabase project is active
- Check that the `DATABASE_URL` environment variable in Render is correct and includes the full hostname
- Ensure your Supabase project allows connections from Render's IP ranges (usually open by default)

### "Insufficient stock" errors on first order
- Run `python migrate.py` in the Render shell to seed books with initial stock of 50 each
